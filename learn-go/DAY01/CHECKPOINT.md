**Closed book.** No looking back at the lesson. Answer in your own words.

1. What does `:=` do that's different from `var`?
   this creates a variable and infers the type

2. What would `fmt.Printf("Age: %d", 30)` print?
   Age: 30

3. Your program has a bug: `go run main.go` gives `"undefined: fmt"`. What's wrong?
   not imported the fmt packages

4. You write `price := 9.99`. What type does Go infer for `price`?
   float64

5. Why would a senior engineer NEVER use `float64` to store a price in a banking app?
   this is an approximate decimal number so could be wrong
