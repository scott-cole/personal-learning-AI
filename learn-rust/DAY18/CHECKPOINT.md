# DAY18: Checkpoint

**Closed book.** No looking back.

1. What attribute marks a function as a test?
   #[test]

2. What does `assert_eq!(a, b)` do?
   checks that a equals b, panics if they're different

3. How do you mark a test that is expected to panic?
   #[should_panic]

4. What does `#[cfg(test)]` do?
   compiles the module only when running tests

5. Where do integration tests live in a Cargo project?
   in the tests/ directory at the crate root
